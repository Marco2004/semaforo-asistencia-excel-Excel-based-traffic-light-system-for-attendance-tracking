# Support / Soporte

## Espanol

## Canales de soporte

- GitHub Issues: para reportar errores, dudas o mejoras.
- Pull Requests: para proponer cambios documentados.
- Documentacion: revisar primero `README.md`, `docs/WORKBOOK.md` y `SECURITY.md`.

## Que incluir en un issue

- Descripcion del problema.
- Pasos para reproducirlo.
- Version de Excel usada.
- Capturas si ayudan, sin datos reales.
- Confirmacion de que el archivo no contiene informacion sensible.

## No publicar

- Datos reales de empleados.
- Registros reales de asistencia.
- Informacion interna de una organizacion.
- Correos, telefonos o datos personales.

## English

## Support channels

- GitHub Issues: for bugs, questions, or improvements.
- Pull Requests: for documented changes.
- Documentation: review `README.md`, `docs/WORKBOOK.md`, and `SECURITY.md` first.

## What to include in an issue

- Problem description.
- Steps to reproduce.
- Excel version used.
- Screenshots if useful, without real data.
- Confirmation that the file does not contain sensitive information.

## Do not publish

- Real employee data.
- Real attendance records.
- Internal organizational information.
- Emails, phone numbers, or personal data.
