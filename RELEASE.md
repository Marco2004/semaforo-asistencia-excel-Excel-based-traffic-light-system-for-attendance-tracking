# Release Process / Proceso de Publicacion

## Espanol

## Checklist de release

1. Confirmar que el workbook demo no contiene datos reales.
2. Abrir el archivo en Microsoft Excel.
3. Verificar formulas, filtros, validaciones y tableros.
4. Actualizar `CHANGELOG.md`.
5. Actualizar `README.md` y `docs/WORKBOOK.md` si cambio el funcionamiento.
6. Revisar `SECURITY.md` si hubo cambios de privacidad.
7. Ejecutar GitHub Actions.
8. Crear tag de version.
9. Publicar release en GitHub.

## Convencion de version

Usar versionado simple:

- `v1.0.0`: primera version estable.
- `v1.1.0`: nuevas funcionalidades o documentacion importante.
- `v1.1.1`: correcciones menores.

## English

## Release checklist

1. Confirm that the demo workbook does not contain real data.
2. Open the file in Microsoft Excel.
3. Verify formulas, filters, validations, and dashboards.
4. Update `CHANGELOG.md`.
5. Update `README.md` and `docs/WORKBOOK.md` if behavior changed.
6. Review `SECURITY.md` if privacy handling changed.
7. Run GitHub Actions.
8. Create a version tag.
9. Publish a GitHub release.

## Version convention

Use simple versioning:

- `v1.0.0`: first stable version.
- `v1.1.0`: new features or important documentation.
- `v1.1.1`: minor fixes.
